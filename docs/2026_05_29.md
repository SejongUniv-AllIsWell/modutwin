## 1. 근본원인 (확정)

### 1.1 Cloudflare 100MB 요청 본문 한도 (commit-final pending 의 진짜 원인)
- `splat.wiki` 는 Cloudflare 뒤 (`nginx/nginx.conf` `real_ip_header CF-Connecting-IP`). **100MB 초과 단일 multipart/fetch 본문은 origin(nginx) 도달 전 Cloudflare 가 드롭** → 브라우저 무한 pending, nginx/백엔드엔 0바이트.
- nginx 는 병목 아님 (`client_max_body_size 0`, nginx.conf:12,158).
- commit-final 이 ~132MB refined PLY 를 단일 multipart 로 전송 → 차단. (적대적 검증으로 "업로드 버퍼링 hang" 가설은 기각, Cloudflare 가 실제 원인으로 확정.)
- ⚠️ **nginx 로그는 컨테이너 내부에서 `/dev/stdout` 심볼릭** — `docker exec ... cat /var/log/nginx/*` 는 항상 빈 것처럼 보임. 반드시 `docker logs 3dgs-nginx` 로 볼 것.

### 1.2 원본 PLY 는 MinIO 에 저장 안 됨 (검증으로 정정된 사실)
- `register-local`(`backend/app/api/uploads.py:781`)은 **placeholder 행만** 생성(바이트 0, file_size=클라이언트값, status=completed). SAM3 `prepare` 는 백엔드 로컬 디스크(`/var/lib/sam3-temp`, 30분 TTL).
- MinIO 에 들어가는 PLY 는 **refined PLY 뿐** (다듬기 중 `/refine/refined-upload-url`, 최종 final.ply 는 commit-final).

## 5. GPU SOG 보류 (호스트 레벨 이슈)

- splat-transform 의 `-g` GPU = **WebGPU(Dawn) = Vulkan** (CUDA 백엔드 없음). `worker/splat-transform` 의 `webgpu` 0.4.0(optionalDependency, rollup external).
- 호스트(DGX): `/usr/share/vulkan/icd.d/nvidia_icd.json` 존재, `nvidia-smi` = **GB10, 드라이버 580.142** → 호스트는 Vulkan 가능.
- 컨테이너: GPU 가 **CDI 방식**으로 주입됨(`docker info` 에 nvidia 런타임 없음, Default=runc). CDI 가 **compute 만** 넣고 **Vulkan graphics 스택은 미주입**.
  - `NVIDIA_DRIVER_CAPABILITIES=all` 설정해도 무시됨(CDI 모드).
  - `libGLX_nvidia.so.0` 는 컨테이너에 있으나, ICD json 을 넣어 로더가 읽어도 `Could not get 'vkCreateInstance' via 'vk_icdGetInstanceProcAddr'` → **libGLX_nvidia 의 Vulkan 의존 lib(libnvidia-glvkspirv 등)가 미주입**되어 ICD 진입점 제공 실패.
- **결론**: 현재 **CPU 변환으로 확정**(동작 확인, training 안 막음). GPU 를 살리려면 **호스트 CDI 스펙을 graphics 포함으로 재생성** 필요 — DGX(관리형 어플라이언스) 호스트 변경이라 신중:
  ```bash
  nvidia-ctk --version
  sudo nvidia-ctk cdi generate --output=/etc/cdi/nvidia.yaml
  # 컨테이너 재생성 후 `docker exec 3dgs-sog-worker vulkaninfo` 로 GB10 enumerate 확인
  ```
  - 검증 시 ICD baked-in 은 단독으론 무효(위 진입점 에러). graphics 스택 전체 주입이 선행돼야 함.
